Overview

Each page introduces one component from the design system while preserving the fixed 1920×1080 slide architecture, typography rules, and editorial art direction.

This file translates the DESIGN.md theme into a browsable HTML component deck.

01 — Card

Card Component

The default card is an editorial content block with a square edge, soft paper fill, left accent rail, and compact hierarchy for icon, label, title, and body copy.

01

Strategic Insight

Cards are designed to present concise narrative content with strong hierarchy and restrained decoration.

02 — Image Card

Image Card

The image card is used when visual content needs a supporting caption. It keeps the sharp editorial frame and places explanatory text in a muted caption band below the image.

Image-based evidence block with a restrained caption treatment.

03 — Card with Image Header

Card Image Variant

This variation combines a top image banner with the standard card structure. It works well for case studies, startup profiles, or feature summaries that benefit from immediate visual framing.

03

Case Profile

04 — Avatar

Avatar Sizes

Avatar components support profile, team, and testimonial layouts. The system includes small, default, and large sizes, each retaining the white ring and minimal shadow treatment.

05 — Stat Card

Stat Card

Stat cards present headline metrics using large condensed numerals, a top gradient bar, and short supporting descriptions. They are intended for KPI, impact, and market summary slides.

08 — Evidence List

Evidence List

The evidence list is a stripped-back editorial bullet system. It replaces conventional bullets with short orange rules to keep lists clean, compact, and consistent with the theme.

  • Supports concise insight summaries without UI-heavy styling.
  • Preserves readability on dense content slides.
  • Uses graphic cues instead of standard bullet symbols.
  • Fits report, analysis, and executive-summary pages.
